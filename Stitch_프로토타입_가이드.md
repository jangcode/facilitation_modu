# [Project A] Stitch를 활용한 프로토타입 제작 가이드

본 문서는 **Stitch** 도구에 직접 생성한 여행 앱 UI 프로젝트의 후가공 및 연결 워크플로우를 안내합니다.

## 1. Stitch 프로젝트 접속
- 이미 **Stitch 내에 새 프로젝트가 생성되었으며**, 3가지 핵심 화면(Home, Search Results, Hotel Detail)이 성공적으로 제너레이션 되었습니다.
- **프로젝트 ID**: `15566429067796604140`
- Stitch(AI UI/UX 툴) 작업 공간을 열고 위 프로젝트에 진입합니다.

## 2. 텍스트 및 UI 후가공 작업
AI가 생성한 UI 내에서 일부 텍스트나 아이콘을 원하시는 대로 다듬습니다.
1. **메인 화면**: 검색창 내부의 텍스트를 "어디로 여행가시나요?" 등으로 덮어쓰기/수정합니다. 하단 네비게이션 버튼들을 "Home", "Saved", "Profile" 등으로 명확히 지정합니다.
2. **검색 결과 화면**: 상단 필터 텍스트를 "Filters", 가격 정보를 실제 통화(예: "$120/night") 형태로 텍스트 오버레이 또는 수정 처리합니다.
3. **상세 페이지**: 호텔 이름과 하단의 커다란 버튼 텍스트를 "예약하기 (Book Now)"로 수정합니다.

## 3. 화면 간 연결 (Hotspot & Navigation)
Stitch 툴의 편집 기능을 사용하여 각 화면을 연결하여 실제 작동하는 것과 같은 화면 전환 흐름을 만듭니다.
- **Home 화면의 '추천 목적지 썸네일' 또는 '검색바' 클릭** → `Search Results` 화면으로 이동하도록 네비게이션 연결.
- **Results 화면의 '호텔 썸네일' 중 하나 클릭** → `Hotel Detail` 화면으로 이동하도록 연결.
- (선택 사항) 상세 화면과 검색 결과 화면에 이전 화면으로 돌아갈 수 있는 뒤로 가기(Back) 버튼 인터랙션을 추가합니다.

## 4. 최종 결과물 제출
- Stitch 상에서 디자인 후가공 및 프로토타입 연결이 모두 완료되면, 해당 **프로젝트의 공유 링크**를 복사하여 최종 산출물로 제출하시면 미션이 완료됩니다!
