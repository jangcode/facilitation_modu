# 소셜 미디어 콘텐츠 A/B 테스트 시나리오 기획

본 문서는 보너스 과제 요구사항에 따른 "톤앤매너 A/B 테스트" 전략을 정의합니다. 동일한 주제를 입력받았을 때, 텍스트의 어조(Tone & Manner)를 2가지 버전으로 생성하여 어떠한 형태가 타겟 오디언스에게 더 효과적인지 비교 분석합니다.

## 1. 테스트 목적
동일한 정보를 전달할 때, **전문적이고 신뢰감 있는 톤(버전 A)**과 **친근하고 유머러스한 톤(버전 B)** 중 어떤 톤이 사용자 참여(좋아요, 리트윗, 체류시간 등)를 더 많이 이끌어내는지 확인합니다.

## 2. A/B 테스트 구성 (예시: 블로그 플랫폼)

### [버전 A] 전문적이고 신뢰감 있는 톤 (Professional & Trustworthy)
- **타겟 오디언스**: 실무자, 기업 임원, IT 정보 탐색자
- **특징**: 객관적 데이터 중심, 정중한 경어체(~습니다), 논리적 전개
- **A버전 전용 프롬프트 지시어 추가**:
  ```text
  - 문체: 객관적이고 신뢰감을 주는 전문가의 말투(~습니다, ~합니다)를 사용할 것.
  - 감정 표현이나 이모지를 최소화하고 수치나 팩트 위주로 논리적으로 전개할 것.
  ```

### [버전 B] 친근하고 트렌디한 톤 (Friendly & Trendy)
- **타겟 오디언스**: 일반 대중, 주니어 직장인, 대학생
- **특징**: 대화체(~해요), 밈(Meme) 활용, 적극적인 이모지 사용, 쉬운 비유
- **B버전 전용 프롬프트 지시어 추가**:
  ```text
  - 문체: 친한 친구에게 설명하듯 편안하고 친근한 대화체(~해요, ~죠)를 사용할 것.
  - 최신 밈이나 유머러스한 비유를 활용하여 딱딱하지 않게 설명할 것.
  - 이모지를 적절히 섞어 시각적인 재미를 줄 것.
  ```

## 3. 워크플로우 내 구현 방안
노코드 툴(Make 등)에서 **라우터(Router)**를 하나 더 분기하여, 랜덤하게 0과 1을 생성(Math 함수)한 뒤:
- **0이 나오면**: 버전 A 프롬프트로 API 호출 -> Notion DB의 '톤앤매너' 컬럼에 `Professional` 기록
- **1이 나오면**: 버전 B 프롬프트로 API 호출 -> Notion DB의 '톤앤매너' 컬럼에 `Friendly` 기록

## 4. 성과 측정 지표 (Metrics)
실제 소셜 미디어 게시 후 다음 지표를 통해 승자를 결정합니다.
- **인스타그램**: 좋아요 수, 공유 수, 저장 수
- **블로그**: 페이지 체류 시간(Avg. Time on Page), 스크롤 뎁스
- **X (트위터)**: 리트윗 수, 북마크 수
