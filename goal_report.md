# 최종 학습 결과 보고서 (goal_report.md)

이 보고서는 노코드 자동화 도구를 직접 활용하며 학습한 자동화 설계의 핵심 개념과 비교 분석 결론을 기술한 성과 보고서입니다.

---

## 1. Trigger와 Action의 개념

- **Trigger (트리거)**:
  - **정의**: 자동화 워크플로우의 '시작점(Start Event)' 역할을 하는 감지 조건입니다.
  - **설명**: 외부 서비스나 데이터베이스에서 특정 사건(Event)이 일어났을 때 이를 감지하여 워크플로우를 가동시킵니다.
  - **예시**: Google Forms에 설문 응답이 제출되는 것, Notion 데이터베이스에 새 페이지가 등록되는 것, Webhook을 통해 특정 데이터 페이로드가 유입되는 것.
- **Action (액션)**:
  - **정의**: 트리거에 의해 워크플로우가 시작된 후, 단계별로 실제 데이터를 '처리 및 수행(Operation)'하는 동작입니다.
  - **설명**: 트리거로부터 전달받은 데이터를 가공하거나, 외부 서비스 API를 호출하여 데이터 등록, 메신저 발송 등의 실질적인 연동 결과를 만들어 냅니다.
  - **예시**: Google Sheets에 새로운 행을 삽입하여 기록하는 것, Slack에 요약 메시지를 전송하는 것, GPT API에 텍스트 요약을 요청하는 것.

---

## 2. 조건 분기 (Filter / Router)의 역할

- **역할 및 가치**:
  - **데이터 흐름 제어**: 유입된 데이터를 검사하여 특정 조건에 부합할 때만 다음 액션을 수행하도록 제한하거나, 데이터 내용에 따라 서로 다른 여러 경로(Route)로 분류합니다.
  - **업무 효율성 및 예외 대응**: 불필요한 연동 실행을 차단하여 자동화 도구의 실행 횟수(Operation/Task)를 절약하고, 비즈니스 규칙(예: 우수 만족도 고객과 개선 제안 고객의 대응 부서 다름)을 자동화 로직 내에 녹여낼 수 있습니다.
  - **예시**: 만족도 점수가 `80점 이상`이면 우수 채널로, `80점 미만`이면 불만/개선 채널로 알림 라우팅.

---

## 3. 자동화 도구의 특징 비교 (Make vs Zapier)

- **Make**:
  - **특징**: 무한한 확장성과 시각적인 드래그 앤 드롭 캔버스를 제공합니다. 무료 플랜에서도 조건 분기와 다중 단계를 제약 없이 사용할 수 있어 매우 경제적입니다.
  - **로그/디버깅**: 각 실행 시점의 데이터 입출력 JSON을 노드별로 실시간 추적할 수 있어 정교한 문제 해결(Troubleshooting)이 가능합니다.
- **Zapier**:
  - **특징**: 수직형 마법사 방식으로 연동을 신속히 완료할 수 있으며, 지원하는 서드파티 서비스의 수가 업계 최대입니다.
  - **제약**: 다중 단계나 조건 분기(Paths) 기능을 활용하려면 유료 요금제 구독이 강제되어 초기 단계에서 비용 허들이 높습니다.

---

## 4. 특정 업무에 적합한 도구 선택 및 이유

- **비용 효율성과 복잡한 로직이 중요한 업무 (추천: Make)**:
  - **이유**: 무료 요금제에서 지원하는 월 1,000 Ops 범위와 무제한 조건 분기는 소규모 조직이나 개인의 복잡한 데이터 파이프라인 설계에 이상적입니다.
- **빠른 업무 도입과 풍부한 통합 라이브러리가 중요한 업무 (추천: Zapier)**:
  - **이유**: 비전문가도 몇 번의 클릭만으로 신속히 워크플로우를 생성할 수 있고, 다루고자 하는 로컬 서비스/틈새 서비스들과의 연동성이 매우 뛰어납니다.

---

## 5. 자동화 흐름 단계별 설명 예시 (Notion 요약 워크플로우)

1. **[감지 단계] Trigger**: Notion에 '2026-06-11 회의록'이 새로 생성되면, 문서 ID와 회의록 본문 텍스트를 감지하여 Make 시나리오를 가동합니다.
2. **[AI 분석 단계] Action 1**: GPT/Gemini API로 원본 텍스트를 전송하여 핵심 요약 및 카테고리(`개발`/`기타`)를 추출하여 JSON 구조로 반환받습니다.
   - *실패 대응*: 이 단계에서 API 서버 오류 발생 시, Error Handler가 트리거되어 Gmail을 통해 관리자에게 예외 메시지를 즉시 알립니다.
3. **[분기 단계] Filter/Router**: AI가 반환한 카테고리 속성값을 기반으로 흐름을 라우팅합니다.
   - `개발` 카테고리인 경우 -> `#dev-team` Slack 채널로 요약 알림 전송.
   - `기타` 카테고리인 경우 -> `#general` Slack 채널로 요약 알림 전송.
4. **[기록 단계] Action 2**: 원래 생성된 Notion 회의록 페이지의 `Summary` 속성 필드에 AI 요약 결과를 업데이트하여 데이터 일관성을 유지합니다.
