# 노션 데이터베이스 구축 가이드

본 문서는 뉴스 자동화 파이프라인에서 결과물이 저장될 노션(Notion) 데이터베이스의 스키마와 속성 설정 방법을 안내합니다. Make.com과 같은 자동화 툴에서 정상적으로 데이터를 주입하려면 정확한 속성 타입이 맞춰져 있어야 합니다.

## 데이터베이스 생성
1. 노션에 접속하여 새로운 페이지를 생성합니다.
2. 페이지 하단의 **데이터베이스 - 인라인(Inline)** 또는 **전체 페이지(Full page)**를 선택합니다.
3. 데이터베이스 이름은 **"IT/AI 기술 동향 스크랩"** 등으로 지정합니다.

## 속성(Properties) 스키마 정의

데이터베이스에 아래와 같은 속성(칼럼)들을 생성해야 합니다. 자동화 툴 설정 시 이 속성들의 이름과 타입이 중요합니다.

| 속성 이름 | 속성 유형(Type) | 설명 (Make에서 매핑할 데이터) |
| --- | --- | --- |
| **제목** | `Title` (제목) | RSS 피드에서 수집된 기사의 원본 제목 |
| **요약문** | `Text` (텍스트) | OpenAI 모듈에서 반환된 3줄 요약 텍스트 |
| **원문 링크** | `URL` (링크) | 기사 원문으로 이동하는 웹 링크 |
| **발행일시** | `Date` (날짜) | RSS 피드 상의 기사 발행 시간 (또는 수집 일자) |
| **중복방지키** | `Text` (텍스트) | (선택 권장) 고유한 URL이나 GUID 값을 저장하여 중복 저장을 방지하기 위해 사용 |

> [!TIP]
> **중복 검사 노하우**  
> Make에서 데이터를 추가(`Create a Database Item`)하기 전에 노션 `Search Objects` 모듈을 배치해 **중복방지키**가 일치하는 데이터가 있는지 검사하도록 구성하면, 동일 기사의 중복 수집을 막을 수 있습니다.

## 노션 API 연동 보안
- 생성된 노션 데이터베이스는 우측 상단의 `...` 메뉴 > **연결(Connections)**에서 Make(또는 통합 플랫폼) 앱을 추가해야만 자동화 툴에서 데이터를 조회 및 기록할 수 있습니다.
- 연동 토큰 관리는 보안 상 중요하므로 하드코딩하지 않고 각 플랫폼의 인증 모듈(OAuth 또는 환경변수)을 통해 안전하게 저장하세요.
