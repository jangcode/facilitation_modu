## [Project A] AI 기반 UI/UX 디자인 시안 제작

### 1. 미션 소개

백 마디 말보다 한 장의 그림이 가진 힘은 강력합니다. 아무리 혁신적인 비즈니스 모델도 텍스트로만 존재할 때는 기획자의 머릿속을 벗어나기 어렵습니다.

훌륭한 아이디어가 '실체가 없다'는 이유로 과소평가되지 않도록, 이제는 상상을 즉시 현실로 끄집어낼 수 있는 도구가 필요합니다.

이번 미션은 단순히 예쁜 이미지를 만드는 법을 배우는 과정이 아닙니다.

생성형 AI를 실무 파이프라인에 이식하여, 기획 의도를 정교한 UI/UX 시안으로 구체화하는 '비주얼 커뮤니케이션' 역량을 확보하는 데 목적이 있습니다.

텍스트 프롬프트로 도출한 시각적 결과물을 피그마(Figma)와 같은 전문 도구와 결합해 실제 작동하는 프로토타입으로 구체화하는 전 과정을 경험합니다.

나아가 '프롬프트 최적화 – 정교한 후가공 – 프로토타입 완성'으로 이어지는 체계적인 AI 협업 워크플로우를 직접 구축하며 실전 감각을 익힙니다.

이 경험은 디자인 전문 기술 없이도 아이디어를 빠르게 시각화하는 실무 역량의 기반이 됩니다.



### 2. 최종 결과물

다음 3가지 산출물을 포함한 AI 기반 UI/UX 디자인 시안 패키지를 팀 단위로 완성한다.


1. UI 디자인 이미지 (3장 이상)

- 서비스의 핵심 기능을 시각적으로 표현한 모바일 또는 웹 UI 디자인 이미지

- 메인 페이지, 목록 페이지, 상세 페이지 등 화면 역할이 구분되는 구성

- 이미지 내 텍스트 깨짐이 수정된 상태

- 파일 형식: PNG 또는 JPG


2. 작업 로그 문서 (1개)

- 디자인 생성에 사용한 프롬프트 기록

- 프롬프트 수정 과정 (초안 → 수정 → 최종)

- 각 수정 단계에서 변경한 이유와 결과 차이 기록

3. Figma 프로젝트 (1개)

- 생성된 이미지를 배치하여 화면 흐름을 구성한 프로젝트 링크

- 클릭 가능한 영역(Hotspot) 또는 화면 전환 흐름 표시


### 3. 과제 목표

이 과제를 마친 후, 학습자는 아래를 스스로 설명할 수 있어야 한다.

- 텍스트 프롬프트의 구성 요소(스타일, 레이아웃, 색상 등)가 결과물에 미치는 영향을 설명할 수 있다.

- AI 생성 이미지의 문제점(텍스트 깨짐, 부자연스러운 요소)을 식별하고 수정하는 방법을 설명할 수 있다.

- AI 생성 이미지의 일관성을 유지하기 위한 방법(시드 고정, 이미지 레퍼런스 등)을 설명할 수 있다.

- 기획 → 이미지 생성 → 후가공 → 프로토타입으로 이어지는 AI 협업 워크플로우를 설명할 수 있다.


### 4. 기능 요구 사항

다음 요구사항을 모두 만족해야 한다.

1. 이미지 생성

- 이미지 생성 AI 도구(Midjourney, DALL-E, Stable Diffusion 등)를 1개 이상 사용한다.

- 최소 3개 이상의 화면(메인 페이지, 목록 페이지, 상세 페이지 등)을 생성한다.

- 생성된 UI 디자인은 모바일(9:16) 또는 데스크탑(16:9)의 일반적인 화면 비율을 준수한다.

2. 프롬프트 최적화

- 프롬프트 개선 과정(초안 → 수정 → 최종)을 문서로 기록한다.

- 각 수정 단계에서 변경한 이유와 결과 차이를 기록한다.

3. 이미지 후가공

- AI가 생성한 이미지 속 뭉개진 글자나 부자연스러운 요소를 수정한다.

- 필요시 인페인팅, 아웃페인팅, 업스케일링 등의 기능을 활용할 수 있다.

4. 프로토타입 제작

- Figma 등 디자인 도구를 사용하여 생성된 이미지를 배치한다.

- 화면 간 이동 흐름을 나타내는 클릭 가능 영역(Hotspot) 또는 화면 전환을 표시한다.


### 5. 보너스 과제 (선택)

1. 보너스 1 – 코드 변환 체험

- 완성된 디자인 시안 이미지를 V0, ChatGPT 등을 활용해 HTML/CSS 코드로 변환해본다.


### 6. 제약 사항

- 도구 사용 및 명시

	- 이미지 생성 도구는 저작권 문제가 해결된 유료/무료 AI 툴을 자유롭게 선택하되 사용한 툴의 이름을 반드시 명시한다.

	- 디자인 편집 및 배치는 Figma 사용을 권장한다(다른 도구 활용 시 사용 이유와 함께 도구명 명시).

- 저작권 및 윤리

	- 레퍼런스 활용 시 출처를 필수로 기록해야 한다.

	- 핀터레스트, Dribbble 등에서 타인의 작업물을 캡처하여 제출하는 행위는 엄격히 금지한다.

- 품질 기준

	- AI가 생성한 이미지 속 뭉개진 글자나 부자연스러운 요소를 수정 없이 그대로 제출해서는 안 된다.
