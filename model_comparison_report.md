# LLM 모델 비교·선정 보고서

## 1. 비교 목적
회의록 요약 및 업무 자동화 파이프라인 구축을 위해, 각 LLM 모델의 성능을 정량적/정성적 기준에 따라 비교 분석하고 업무에 가장 적합한 최적의 모델을 선정하고자 합니다.

## 2. 사용 환경 및 조건 기록 (재현성)
- **비교 분석 수행일**: 2026-06-11
- **비교 대상 모델 및 사용 환경**:
  1. **GPT-4o**: 웹 인터페이스 (ChatGPT Plus 요금제) / Temperature: 기본값
  2. **Claude 3.5 Sonnet**: 웹 인터페이스 (Claude Pro 요금제) / Temperature: 기본값
  3. **Gemini 1.5 Pro**: Google AI Studio (API 및 웹 Console) / Temperature: 0.4, System Instruction 적용 가능

## 3. 비교에 사용한 동일 입력 (테스트 프롬프트)
```markdown
[업무 과업] 회의록 요약

[원문]
(B1-1/회의록.md의 '수입 주류 진품인증 마그네틱 씰 공식 인정 관련 협의 회의록' 전문 입력)

[원하는 결과] 
- 최종 결론 요약 (3문장 이내)
- 결정사항 (불릿 형태)
- Action Items (담당자, 기한, 세부 작업 포함)

[톤] 격식 있고 간결한 사내 공유용 보고서 톤
[금지 사항] 추측성 문장 작성 금지, 회의록에 등장하지 않은 인물이나 날짜 언급 금지.
```

## 4. 평가 축 및 점수표
평가는 각 항목당 1점(매우 미흡) ~ 5점(매우 우수) 척도로 진행하였습니다.

| 평가 축 (Criteria) | 설명 | GPT-4o | Claude 3.5 Sonnet | Gemini 1.5 Pro |
| :--- | :--- | :---: | :---: | :---: |
| **1) 정확성 (사실 반영)** | 회의록 내 사실(날짜, 인물명, 직책 등) 관계를 왜곡이나 누락 없이 정확하게 파악하는지 | 5 | 5 | 5 |
| **2) 한국어 자연스러움** | 번역 투나 어색한 한자어를 배제하고 자연스러운 비즈니스 한국어로 요약하는지 | 4 | 5 | 4 |
| **3) 문맥 유지력** | 10턴 이상의 대화 중 지시 사항 변경(예: 실명 블라인드 처리)을 일관되게 기억하는지 | 4 | 5 | 5 |
| **4) 형식 준수성** | 요구한 출력 규격(템플릿 구조, 불릿 등) 및 금지 규칙을 완전하게 지키는지 | 5 | 5 | 4 |
| **평균 점수** | | **4.5** | **5.0** | **4.5** |

## 5. 모델별 결과 요약 및 평가 근거

### GPT-4o
- **정확성 & 형식 준수**: 지정한 출력 규격을 매우 정확하게 준수하며 마크다운 형식을 깔끔하게 처리함.
- **한국어 자연스러움**: 전반적으로 우수하나, 간혹 명사형 종결 어미가 어색하게 연결되는 번역체 스타일이 남아 있음.
- **문맥 유지력**: 대화 턴수가 늘어날 때 초반에 언급했던 세부 금지 규칙(예: 실명 사용 금지)이 간혹 누락되는 양상을 보임.

### Claude 3.5 Sonnet
- **정확성 & 한국어 자연스러움**: 문맥의 흐름을 이해하고 한국어 요약문을 문맥상 가장 매끄럽고 고급스러운 비즈니스 어조로 생성하는 데 독보적인 성능을 보임.
- **문맥 유지력**: 시스템 프롬프트에 작성한 제약 조건을 대화가 진행되는 동안 일관되게 기억하고 반영함.
- **형식 준수성**: 복잡한 Few-shot 규칙과 되묻기 흐름을 가장 직관적으로 이해하고 수행함.

### Gemini 1.5 Pro
- **문맥 유지력**: 긴 컨텍스트 윈도우 덕분에 10턴 이상의 복잡한 상황 변화 속에서도 정보의 일관성을 가장 단단하게 유지함.
- **한국어 자연스러움**: 요약 품질은 준수하지만, 가끔 지나치게 직역에 가깝거나 건조한 어투를 사용하여 추가 튜닝이 필요했음.
- **형식 준수성**: 가끔 금지 조건을 우회하거나 템플릿의 소제목을 임의로 생략하려는 경향이 미세하게 발견됨.

## 6. 최종 선정 결론
본 '회의록 요약 및 정보 관리' 과업을 수행할 최적의 모델로 **Claude 3.5 Sonnet**을 선정합니다.

**선정 근거**:
1. 회의록 내용을 요약하는 과정에서 격식 있는 한국어 보고서 톤을 가장 매끄럽고 일관되게 구성하여, 사람이 재검토하고 수정해야 하는 시간을 가장 많이 단축시킵니다.
2. Few-shot에 포함된 복잡한 시나리오(모호한 정보 유입 시 되묻기 기능 등)를 추가 오작동 없이 정확하게 학습하고 실행하는 능력이 가장 뛰어납니다.
3. 10턴 이상의 대화에서도 시스템 프롬프트에서 설정한 규칙(실명 사용 금지, 추측성 표현 배제)을 어기지 않는 안정적인 통제 성능을 보여주었습니다.
