# 미션 3: 노코드 자동화 기초 - 워크플로우 설계

## 과제명
미션 3: 노코드 자동화 기초 - 워크플로우 설계

## 요구사항 정리

### 1. 공통 요구 사항 (동작 및 보안)
*   **실제 동작하는 워크플로우 구현**: 개념적인 설계에 그치지 않고, 직접 툴 상에서 동작 가능한 워크플로우를 생성해야 합니다.
*   **워크플로우 최소 구조**:
    *   Trigger 1개 이상
    *   Action 2개 이상
    *   조건 분기(Filter 또는 Router) 1개 이상
*   **분기 경로 실행 검증**: 조건 분기에 의해 나뉘는 모든 경로가 실제로 최소 1회 이상 정상 작동했음을 확인하는 실행 결과가 필요합니다.
*   **보안성 유지**: 제출하는 스크린샷, 보고서, 공유 링크 등에 API Key, 토큰, 비밀번호 등 민감한 개인 정보가 노출되지 않도록 마스킹(예: `***`) 및 가림 처리를 해야 합니다.

### 2. [프로젝트 1] 자동화 도구 비교 구현 요구 사항
*   **2개 이상의 도구 사용**: 동일한 시나리오의 워크플로우를 서로 다른 2개 이상의 자동화 툴(예: Make, Zapier 등)로 각각 구축합니다.
*   **비교 분석 보고서 작성**: Markdown 형식으로 작성하며 다음 사항들을 필수로 포함해야 합니다.
    *   사용 도구 명칭 및 구현 과정 요약
    *   최소 5개 이상의 비교 항목 (예: UI/UX, 설정 난이도, 연동 서비스 범위, 무료 플랜 혜택, 실행 로그 및 디버깅 방식 등)
    *   각 도구의 구체적인 장단점 및 적합한 상황에 대한 퍼실리테이터 의견
*   **시각 자료**: 각 도구별 워크플로우 구성 화면과 실제 실행 성공(Success Log) 결과 화면 캡처본이 포함되어야 합니다.

### 3. [프로젝트 2] 자유 주제 자동화 설계 및 구현 요구 사항
*   **반복 업무 정의**: 매일 또는 주기적으로 발생하는 비효율적인 반복 업무 중 자동화할 대상을 1개 정의합니다.
*   **도구 선정 및 이유**: 비교 분석을 토대로 가장 적합하다고 판단한 1개의 자동화 도구를 선정하고 그 구체적인 이유를 밝힙니다.
*   **자동 실행 흐름 구현**: Trigger가 감지되면 어떠한 수동 조작 없이 완전히 자동으로 실행되는 구조를 구축합니다.
*   **설계 문서 작성**: 워크플로우 흐름에 대한 상세 설명 또는 흐름도를 작성합니다.
*   **시각 자료**: 구현 화면과 실제 자동으로 트리거되어 성공한 실행 결과 캡처본이 포함되어야 합니다.

### 4. 보너스 요구 사항 (선택)
*   **AI 연동 Action 추가**: OpenAI(ChatGPT), Claude 등 생성형 AI를 Action 단계에 포함시켜 텍스트 요약, 분류, 메시지 템플릿 제작 등을 자동화합니다.
*   **에러 헨들링 및 알림**: 실행 도중 실패했을 때 이메일/메신저로 즉시 알림을 주도록 설계하고, 재시도 또는 대체 경로를 1개 이상 구성합니다.
