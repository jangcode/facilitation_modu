## 과제명
미션 3: 노코드 자동화 기초 - 워크플로우 설계

## 요구사항 정리
### 1. 기능적 요구사항
- **[공통] 실제 동작하는 자동화 워크플로우 구현**
  - Trigger 1개 이상 포함
  - Action 2개 이상 포함
  - 조건 분기(Filter/Router) 1개 이상 포함
  - 조건 분기 경로가 실제로 1회 이상 실행된 결과를 확인할 수 있는 실행 결과 제공
- **[프로젝트 1] 자동화 도구 비교 구현**
  - 동일한 워크플로우 구조를 서로 다른 2개 이상의 자동화 도구(예: Make, Zapier 등)로 구현
  - 각 도구별 워크플로우 구성 화면 캡처 이미지 제공
  - 각 도구별 실행 결과 화면 캡처 이미지 제공
  - 비교 분석 보고서(Markdown) 작성:
    - 사용한 도구 이름
    - 구현 과정 요약
    - 최소 5개 이상의 비교 항목 (UI/UX, 설정 난이도, 연동 서비스 범위, 무료 플랜 범위, 실행 로그 확인 방식 등)
    - 각 도구의 장단점 정리
    - 상황별 적합한 도구 선택에 대한 의견
- **[프로젝트 2] 자유 주제 자동화 설계 및 구현**
  - 자동화할 반복 업무 1개 정의
  - 자동화 도구 1개 선정 및 선정 이유 작성
  - 워크플로우 설계 문서(설명 또는 다이어그램 포함) 작성
  - 자동 실행 구조 구현 (Trigger 발생 시 수동 개입 없이 자동 실행되어야 함)
  - 구현 화면 캡처 이미지 제공
  - 실행 결과 화면 캡처 이미지 제공
  - 워크플로우 흐름 설명 포함 (단계별 입력/출력 포함)
- **[보너스 과제 (선택)]**
  - AI 연동 Action 추가 (생성형 AI를 통한 자동 텍스트 생성 등)
  - 실패 알림 및 재시도 전략 설계 (실행 실패 시 이메일/메신저 알림, 대체 경로 설계 등)

### 2. 비기능적 요구사항(제약조건)
- **보안성**: 제출물(스크린샷, 문서 등)에 API Key, 토큰, 비밀번호, 민감한 개인정보가 노출되지 않도록 마스킹 처리(예: `***`) 및 가림 처리 필수.
- **실전성**: 두 프로젝트 모두 설계 문서뿐만 아니라 "실제 동작하는 워크플로우"여야 함.
- **자동 실행**: 프로젝트 2는 Trigger 발생 시 실시간/자동 실행되어야 함.
- **과금 리스크 최소화**: 무료 플랜 범위 내에서 완수 가능한 조합 우선 고려.
