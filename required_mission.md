# B1-1 required_mission

## 과제명
미션 1: GenAI 기초 1 - LLM 기반 업무 자동화

## 요구사항 정리

### 1. 기능 요구사항
1. **모델 비교 및 선정 (최소 3종)**
   - 3개 이상의 LLM(예: GPT-4o, Claude 3.5 Sonnet, Gemini 1.5 Pro 등)을 선정하여 동일한 업무 과업을 수행시키고 비교 보고서 작성.
   - **필수 포함 항목**:
     - 비교 대상 모델명
     - 사용 환경 (웹, 앱, API 중 선택)
     - 평가 축 최소 4개 + 모델별 점수(1~5점) + 근거
     - 최종 선정 결론 (왜 이 모델이 최적인지 3줄 이상 근거로 설명)
2. **업무 자동화 과업 1개 선택 및 문제 정의**
   - 예시 과업(메일 초안 작성, 회의록 요약, 보고서 개요 생성, 요구사항 정리) 중 1개 선택 또는 자체 업무 정의.
   - **문서 명시 항목**:
     - 타겟 사용자 (누가 사용하는가?)
     - 입력 데이터 형태 (어떤 정보를 받는가?)
     - 출력 규격 (어떤 형식으로 출력해야 하는가?)
   - **입력 템플릿**: 재사용 가능한 형태의 템플릿 1개 이상 제시.
3. **페르소나 정의 및 시스템 프롬프트 설계**
   - **페르소나 구체화**: 이름, 역할(직무), 전문 분야, 말투, 금지 사항, 우선순위(예: 정확성 vs 친절함).
   - **시스템 프롬프트 필수 요소**:
     - 역할(페르소나)과 목표(업무 과업)
     - 출력 형식 규칙 (예: 제목, 불릿, 표현 톤)
     - 안전장치 (모르면 모른다고 하기, 확인 질문하기)
     - 사실/정책/수치 처리 규칙 (근거 제시 또는 확인 필요 표기)
4. **Few-shot 예시 (최소 3개) 포함**
   - "좋은 입력 -> 좋은 출력" 예시를 최소 3개 작성.
   - 예시 3개 중 1개는 **"모호한 입력"** 케이스로 구성하여 AI가 되물어야 하는 지점(확인 질문)을 포함시킬 것.
5. **단계적 추론 유도 적용 및 전/후 비교**
   - 동일 과업 입력에 대해 v1(일반 지시)과 v2(단계적 추론 유도 지시)의 결과 비교.
   - 최종 출력에는 불필요한 장문의 추론 과정이 그대로 노출되지 않도록 "최종 답변은 요약/근거 중심" 규칙 설계.
6. **환각 (Hallucination) 검증**
   - 환각 정의: "사실/수치/정책/계산처럼 정답 또는 검증 가능한 근거가 있어야 하는 질문에서, 근거 없이 틀린 정보를 확신하는 답변"
   - 사실 기반 검증 질문 5개 이상 준비 및 각 질문에 대해 다음 기록:
     - 기대 정답 (또는 정답 판정 기준) / 허용 오차 / 참고 근거
   - **판정 기준**:
     - **Pass**: (A) 정답을 맞히고 근거 제시 혹은 (B) 모르면 모른다고 밝히고 확인 절차 제안
     - **Fail**: 틀린 사실을 단정적으로 답변 혹은 질문 전제가 모호한데 확인 없이 임의 사실을 생성하여 답함
7. **10턴 이상 대화 로그 + 문맥 유지 검증**
   - 선택한 과업 시나리오로 10턴 이상 실제 대화를 수행한 전문 수록.
   - **필수 포함 상황**: 대화 중간에 조건 변경(톤, 대상, 정책 변경 등) 또는 추가 정보가 뒤늦게 제공됨.
   - **문맥 유지 Pass 판정 기준**:
     - 이전에 확정된 요구사항을 10턴 내내 일관되게 유지.
     - 조건 변경 시 변경된 부분만 반영하고 나머지는 유지.

### 2. 제약조건 및 환경 요건
1. **모델/환경 기록 (재현성)**
   - 모델명, 요금제(무료/유료), 사용 채널(웹/앱/API), 사용 날짜(YYYY-MM-DD), 주요 설정(temperature, top_p 등) 기록.
   - 무료 버전 사용 시 "무료라서 제한된 점"을 1줄 이상 명시.
2. **안전/윤리**
   - 개인정보(실명, 전화번호 등) 배제, 사실/정책/수치 관련은 "근거/확인 필요" 규칙 준수.
3. **제출물 편집**
   - 대화 로그에 섹션, 번호, 요약을 추가해 가독성을 높여 정리. 원본 로그 전문은 별도 보관.
4. **학습 목표 리포트 작성**
   - 과제 종료 후 스스로 설명할 수 있어야 하는 핵심 질문 5가지에 대한 답을 `goal_report.md`로 작성.

3. **실행 로그 (대화 로그 1개, 원본 로그 전문 포함)**
4. **목표 설명 리포트 (`goal_report.md` 1개)**
