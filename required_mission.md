## 과제명
[Project B] 뉴스 요약 자동화 워크플로우 만들기

## 요구사항 정리
1. **데이터 수집**: 
   - 매일 설정된 시간에 스케줄 트리거 작동.
   - RSS 피드를 통해 기술 뉴스를 자동 수집.
   - 사전에 정의된 주제(예: AI)에 해당하는 기사 1건을 선택하여 제목, 링크, 본문 내용 추출.
2. **AI 가공**: 생성형 AI 툴(OpenAI API 등)을 활용하여 뉴스 내용을 3줄 이내로 요약.
3. **자동 저장**: 노션 등 협업 도구의 데이터베이스에 제목, 요약문, 원문 링크, 발행일시를 올바른 속성에 매핑하여 자동 저장.
4. **안정성**: 사람의 개입 없이 프로세스 완료 및 오류/수집 실패 시 예외 처리 적용.
5. **비용 및 중복 방지**: 
   - 중복 저장 방지 키(예: URL 해시, GUID) 적용 권장.
   - 요약 호출은 기사 1건당 1회, 재시도는 최대 2회 상한.
6. **보너스 요건(선택)**: 썸네일 이미지 자동 생성, 감성 분석 태그 추가.
7. **제약 사항**:
   - 도구 명시 (Make, Zapier, n8n 등) 및 저장 도구(노션 권장) 명시.
   - 저작권 침해 크롤링 금지 및 무한 루프 방지.
   - API 키 및 토큰 노출 방지(환경 변수 사용, 문서 내 마스킹).

## 산출물 목록
1. **자동화 워크플로우**
   - 워크플로우 구조를 보여주는 스크린샷.
   - 각 단계별 역할과 연결 구조를 설명하는 문서.
2. **노션 데이터베이스 페이지**
   - AI가 요약한 텍스트가 포함된 뉴스 콘텐츠 데이터 (제목, 요약문, 원문 링크, 발행일시 속성 포함).
3. **README (또는 기획서)**
   - 팀 역할/개인별 작업 요약.
   - 주제 필터링 기준(키워드/태그 목록)과 선택 이유.
   - 에러 처리 정책과 선택 이유.
